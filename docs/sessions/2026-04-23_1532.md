## Achados
- **Bug**: commit 7532224 (lazy-load de pandas/requests pra cortar 530ms
  do boot) removeu `MARKETS` e `_conn` do top-level do launcher. Adicionou
  `__getattr__` de módulo mas **só pra `MAIN_GROUPS`** — qualquer outro
  atributo cai em `AttributeError`. `launcher_support/screens/strategies.py:50`
  acessava `_launcher_mod.MARKETS.get(...)` diretamente → crash silencioso
  em Tk callback → `_clr()` limpava a tela e `render()` morria antes de
  pintar o picker → "aperto BACKTEST e não aparece nada".
- **Sintoma enganoso**: como o `_clr()` roda primeiro (limpa main frame
  + unpack screens_container), a tela vai a branco antes do crash. Parece
  freeze ou button dead, mas é `AttributeError` engolido no tkinter handler.
- **Commit message do 7532224 prometia** "Zero @patch('launcher.ConnectionManager')
  or MARKETS in tests (verified via grep) — safe for test mocking". O grep
  foi restrito a `tests/` mas não varreu `launcher_support/`. Aprendizado:
  quando refatorar module-level symbol, grepar TODO o repo (incluindo
  launcher_support/, api/, bot/, analysis/, tools/) antes de considerar safe.
- **Log do launcher não ajudou a diagnosticar**: `strategies` screen é
  legacy, não emite `paint.*` nem `screen_switch` events. Ausência de
  log não distinguiu "click não fired" de "render crashou". Vale
  adicionar um `emit_timing_metric("paint.strategies", ...)` no final de
  strategies.render — mas não prioridade.
