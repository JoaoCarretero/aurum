## Achados

**Bugs caçados nesta sessão:**

1. **`cmd.exe` piscando no boot.** `subprocess.Popen(ssh)` no Windows
   sem `CREATE_NO_WINDOW` + `STARTF_USESHOWWINDOW` faz janela console
   flashar a cada spawn do watchdog. Fix em `ab10178`.

2. **Badge TUNNEL mostrava `—` apesar do tunnel estar UP.** Problema
   clássico de `__main__` dup: `python launcher.py` registra módulo
   como `__main__` em sys.modules; `from launcher import X` de outro
   arquivo carrega launcher.py DE NOVO como módulo `launcher` com
   singleton `None`. Fix em `43ff0f4` — singleton mora em
   `launcher_support/tunnel_registry.py`, sempre o mesmo módulo
   independente de como launcher foi invocado.

3. **"Clicou ENGINES travou tudo".** HTTP sync no UI thread: quando
   `_render_shadow_panel` chamava `client.latest_run` + `get_heartbeat`,
   TkInter congelava 2-5s (timeout). Fix em `acc067f` — refactor pro
   ShadowPoller em thread daemon que cachea; UI lê cache instantâneo.
   Pattern padrão pra UI responsiva.

4. **`after_handle` leak.** `_schedule_shadow_refresh` appendava um id
   a cada 5s em `after_handles` sem cancelar o anterior. 720
   handles/hora vazando até cleanup. Fix em `4c3e864` — single slot
   `shadow_refresh_aid` com cancel antes de schedule.

**Gaps de deploy descobertos (ver session 0940 — backlog):**
- installer VPS não instala `fastapi/uvicorn` automaticamente
- Crash-loop limit (StartLimitBurst=5) protege bem mas força
  `systemctl reset-failed` após fix de deps

**Aprendizados arquiteturais:**
- Singleton em `launcher_support/*_registry.py` evita armadilhas de
  __main__ dup quando o arquivo é entry point.
- Qualquer HTTP no cockpit deve ir via poller em background thread —
  UI thread tem que ser 100% não-bloqueante.
- Fresh subprocess no Windows precisa flags hide explícitas (não é
  default do Python).
