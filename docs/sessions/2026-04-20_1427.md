## Achados

- **Anti-pattern confirmado**: 50 `.destroy()` + 10+ sites de "destroy
  all children then rebuild" no launcher.py. Cada switch custa ~100-300ms
  de layout solver. ScreenManager com cache elimina isso pra screens
  migradas (sub-ms no cache hit).
- **`_clr()` é leak através de screen migrada**: legacy navigation
  (sem passar pelo ScreenManager) não chamava `on_exit` nas screens
  migradas, deixando timers vivos. Fix: `_clr` agora invoca on_exit na
  current migrated screen antes de rebuild do legacy body.
- **`screens_container` não pode ser filho de `self.main`**: `_clr`
  destrói todos os children de `self.main`. Solução: `screens_container`
  é **sibling** de `self.main` (direct child do Tk root). `_clr` default
  flip pra LEGACY mode (self.main packed, screens forgot); wrapper de
  migrated flip pra MIGRATED mode antes do show().
- **Flake Tk no Windows+Python3.14**: recriar `tk.Tk()` entre tests
  falha com `TclError: invalid command name "tcl_findLibrary"`.
  Mitigação: fixtures `scope="module"` em integration/regression tests.
  Unit tests usam mock Screen (sem Tk) pra evitar.
- **Race com Codex em paralelo (contínua)**: durante esta sessão, Codex
  rodou em paralelo committando paper/cockpit/design doc (7 commits
  entre os meus). Nenhuma colisão com meus paths. CLAUDE.md já tem
  regras pra isso.
