## Achados
1. **pytest-xdist sweet spot é `-n 6`** (não `-n auto`=12 workers) —
   spawn overhead + test I/O contention. Empírico: -n 6 = 26s, -n auto = 36s,
   sequential = 55s.
2. **41 tk-dependent tests skipam sob xdist** (workers sem display Tk).
   Decisão: `-n 6` é opt-in (não default em addopts) pra preservar
   coverage em run sequential.
3. **Lazy `core.data.connections` foi cascade maior que esperado** —
   `MAIN_GROUPS` top-level consumia MARKETS, precisou virar lazy também
   via module-level `__getattr__` (PEP 562). Impacto positivo: launcher
   import 829ms → 149ms (5.5x faster).
4. **`_dash_home_render`, `_dash_portfolio_render`, `_dash_trades_render`,
   `_dash_paper_edit_dialog` já eram thin delegates** de extração
